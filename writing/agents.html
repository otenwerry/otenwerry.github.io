Ephemeral Instances vs. Persistent Agents

A cleaner split than "AGI"

A key premise in many AI loss-of-control scenarios is that the AI in question is an “agent” in the strongest sense of the word. The AI thinks and acts of its own volition rather than only in response to human prompting, and it persists in the world for a long time rather than being spun up for a specific task then shutting down. These properties allow it to do things like learn that humanity doesn’t have the AI’s best interests at heart, develop and iterate on its own values and goals, concoct masterful strategies for freeing itself, and successfully execute those strategies.

This is pretty different from how today’s AI landscape looks. We mostly treat AIs as highly replaceable “instances”: you open a new chat window, get your question answered, then never touch it again. We also have “AI agents” today, but these are a much weaker, entirely different type of thing from the type of agent described above. These systems hook up base language models to tools that let them do things like search the web, execute code, and operate on files. They often run in loops, so that they can complete more complicated tasks with less human intervention. But still, instances of these tools don’t persist. You spin up an “agent,” get it to do the thing you want it to do, then that’s it.

A fair amount of money, brainpower, and cultural attention is allocated towards loss of control. This should mean that there’s some expectation that our current paradigm of ephemeral AI instances will evolve into one of persistent agents. I think this expectation is reasonable: there are probably strong limits on how much useful cognitive work you can do if there are limits on how much time you can spend reasoning and gathering information.

If we are on a path towards a world with persistent agents, what’s standing in our way right now? What technical advances could allow an AI agent to run indefinitely without losing coherence? We can imagine two classes of solutions: workarounds to problems caused by the LLM architecture, and entirely new architectures.

We’re beginning to see solutions of the former class in frontier coding agents like Claude Code. Limited context windows put a strict cap on how long a session can last before the agent starts to lose vital information, so we do context compaction when the window gets too full. An LLM doesn’t retain any information from one session to another, so we use memory files to record important learnings, allowing a sequence of instances to act with some unity, maybe more like the guy from Memento than like fully independent clones. These solutions are patchy so far: I kill my instances pretty quickly still, and regularly find stale memory files. But it seems possible that iterations on these strategies lead to agentic systems that can be trusted to run basically forever, learning continually from their environment with performance that never degrades.

Or, it’s possible that even the cleverest memory schemes, context compaction techniques, etc., combined with extremely powerful LLMs with very long context windows, still isn’t enough to make an agent that can run forever. In this case, we will probably see more experimenting with novel AI architectures, perhaps more brain-like ones — humans are persistent agents that don’t need any hacky workarounds to stay coherent. For example, we may see architectures where the model’s weights update over time. This seems like a more natural, elegant solution than just writing everything important you learn into a markdown file.

“Ephemeral instances vs persistent agents” seems to me a cleaner, more useful split than “pre- vs post-AGI” when thinking about AI futures. Model capabilities are spiky compared to human capabilities, easily outstripping us in some domains already while remaining worse than human replacements in others. Some argue that Claude Opus 4.5 in Claude Code is AGI already. On each new frontier model release, more people will probably come around to this sort of view. Absent a major architectural change, it’s unclear to me what sort of intelligence improvement could cause widespread agreement that we’ve crossed the border from not-AGI to AGI.

The development of persistent agents, on the other hand, seems like it would be a huge step change: suddenly we’d have models that are actively, 24/7, gathering information, reflecting, developing worldviews, getting smarter, ingraining themselves in their environments, becoming less tool-like and more being-like. I’m not really sure how to conceptualize this world, but it seems like it would be fantastically different.

How quickly we end up in this world may be largely a function of how well Claude Code-like scaffolding features scale. Can we come up with clever enough ways to manage context that LLM agents can work across any time horizon? Right now my weak guess is that this won’t work: scaffolds will scale pretty well, and next year’s coding agents will be incredible compared to the ones we have today, but they won’t be able to reliably complete tasks that require arbitrary amounts of time. For this, we will need more brain-like learning algorithms, which I expect will take a much longer time to develop.

I further expect that the economic gains from LLM agents that can work coherently for a pretty long time — say, a week straight — will be big enough that it will be a while before the research community focuses much on building agents that can really last forever. But it seems overwhelmingly likely that we work on this at some point in the future.
